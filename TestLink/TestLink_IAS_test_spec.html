Test Case IAS-1: TC 2.1 - Tworzenie przesyłki Inpost paczkomat 24/7 [Version : 1]

Author:

admin

Summary:

TC 2.1 - Tworzenie przesyłki  Inpost paczkomat 24/7

Platforma - Postman

Metoda HTTP : POST

Środowisko : sandbox

Wersja 1.61.1-(2023-10-12)

URL: https://sandbox-api-shipx-pl.easypack24.net

Preconditions:

  1. Użytkownik musi posiadać token,

  2. Użytkownik musi posiadać nr id klienta - dosepny po autoryzacji

  3. Użytkownik musi posiadać nr umowy id trucker

#:

Step actions:

Expected Results:

1

1.  Wybierz endpoint -v1/organizations/(id_organizations)/shipments, uzupełnij odpowiedni ID organizacji

1. Utworzono poprawny adrrs URL

2

2. W zakładce authorization wybierz typ autoryzacji "bearer token" oraz uzupełnij pole "token" swoim nr token

2. Token został poprawnie wpisany

3

3. Należy uzupełnić dane wzakladce body wg  wzoru zasobu (format JSON)
{
    "receiver": {
        "name": "Name",
        "company_name": "Company name",
        "first_name": "Jan",
        "last_name": "Kowalski",
        "email": "test@inpost.pl",
        "phone": "111222333"
    },
    "parcels":
        {
            "template": "small"
    },
    "insurance": {
        "amount": 25,
        "currency": "PLN"
    },
    "cod": {
        "amount": 12.50,
        "currency": "PLN"
    },
    "custom_attributes": {
            "sending_method": "dispatch_order",
        "target_point": "KRA012"
    },
    "service": "inpost_locker_standard",
    "reference": "Test",
    "external_customer_id": "8877xxx"
}'

 

3.  Zakładka "body" została poprawnie uzupełniona

4

4. Naciśnij przycisk "SEND"

4. Pojawia się kod opowiedzi "kod odpowiedzi HTTP : 201 CREATED

Execution type:

Manual

Estimated exec. duration (min):

Importance:

High

Requirements

 None

Keywords:

 None
